Глоссарий
=========

.. glossary::
   :sorted:

   Микросервис
      Автономный сервис, отвечающий за определённую функциональную область платформы (например, управление моделью данных, процессами, уведомлениями). Микросервисы взаимодействуют между собой через RabbitMQ и REST API.

   Артефакт
      Конфигурационный объект платформы (тип данных, форма, журнал, процесс,
      дашборд, шаблон уведомления и т.д.), хранящийся в формате JSON, YAML,
      XML или ZIP. Артефакты обеспечивают переносимость настроек между средами.

   Атрибут
      Свойство объекта в модели данных Citeck. Атрибуты описывают структуру типа данных и используются для хранения, отображения и обработки информации.

   Тип данных
      Описание структуры объекта в системе, включающее набор атрибутов, связанные формы, журналы и бизнес-процессы. Типы данных являются основой модели данных Citeck.

   Журнал
      Табличное представление данных определённого типа с возможностью фильтрации, сортировки, группировки и настройки отображаемых колонок.

   Форма
      Визуальное представление для создания и редактирования объектов определённого типа данных. Формы конструируются в визуальном редакторе на базе Form.io.

   Дашборд
      Настраиваемая панель, состоящая из виджетов, отображающих сводную информацию по объекту, типу данных или рабочему пространству пользователя.

   Бизнес-процесс (BPMN)
      Автоматизированный процесс, описанный в нотации BPMN 2.0 и выполняемый процессным движком на базе Camunda. Включает пользовательские задачи, сервисные задачи, события и шлюзы.

   Records API
      Универсальный API платформы для работы с данными. Позволяет выполнять CRUD-операции, запрашивать атрибуты объектов и выполнять поиск по различным источникам данных.

   Рабочее пространство (Workspace)
      Изолированная область в интерфейсе Citeck, объединяющая журналы, дашборды и настройки для определённой предметной области или роли пользователя.

   Конечная точка (Endpoint)
      Артефакт, описывающий параметры подключения к внешней системе (URL, протокол, настройки аутентификации). Используется при интеграции с внешними сервисами.

   Секрет (Secret)
      Артефакт для безопасного хранения учётных данных (логин, пароль, токен), используемых при подключении к внешним системам через конечные точки.

   Шаблон уведомления
      Артефакт, определяющий содержимое и формат уведомления (тема, тело письма, получатели). Поддерживает подстановку данных из контекста объекта или процесса.

   DMN
      Decision Model and Notation — стандарт для описания таблиц принятия решений. В Citeck DMN-таблицы встраиваются в BPMN-процессы для автоматизации бизнес-логики.
