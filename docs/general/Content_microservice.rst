Content микросервис
=========================

.. note::

    Доступно только в Enterprise версии.

**Ecos-content** - микросервис, предназначен для обеспечения хранения файлов в системе в определенное файловое хранилище. Микросервис позволяет настроить конфигурацию системы таким образом, чтобы Citeck ECOS имел возможность взаимодействовать одновременно с несколькими файловыми хранилищами и обеспечивать возможность сохранения разных типов объектов системы в разные хранилища.

.. tip::

    1. Обычная практика - менять хранилище глобально через :ref:`настройку конфигурации<content_new_config>`. Если в каком-то специфичном кейсе требуется изменить хранилище для :ref:`конкретного типа на s3<type_content_storage>`, то хранилище также необходимо поменять у системного типа **temp-file**.
    
    2. Если планируется использование системы как хранилище контента и массовая загрузка файлов, то рекомендуется использовать внешнее хранилище s3.
    
    3. На стандартной форме **doclib-file** настроен ряд параметров для корректной загрузки файлов. Если будет использоваться кастомная форма, то нужно учитывать настройки этой формы, в частности обязательности контента и задание конфигурации максимального размера файла на форме. См. :ref:`подробно<new_document_library>`
