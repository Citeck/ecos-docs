Для улучшения навигации и работы в Citeck реализовано разделение данных, пунктов меню и открытых вкладок по **рабочим пространствам (workspace)**.

**Рабочее пространство** — отдельная область в Citeck для совместной работы пользователей над определённой функциональностью. Например, пространство «Договоры и доверенности», в котором пользователи, имеющие доступ, работают с договорами, доверенностями и общесистемными справочниками. Или пространство конкретного проекта, в котором команда работает с тикетами и запросами на изменение — не пересекаясь с данными других проектов.

Поддерживается два типа рабочих пространств:

- :ref:`Персональное<ws_personal>` — индивидуальная область для управления задачами, документами и участия в бизнес-процессах.
- :ref:`Совместное<ws_collaborative>` — коллективная область для командной работы: объединяет участников, документы, задачи и бизнес-процессы в едином интерфейсе.

Рабочее пространство может быть **публичным** (любой пользователь платформы может к нему присоединиться) или **приватным** (доступно только выбранным участникам).

Участники рабочего пространства имеют одну из двух ролей:

- **Пользователь** — работает с данными пространства в рамках предоставленного доступа.
- **Менеджер** — дополнительно управляет настройками пространства: меню, составом участников, артефактами. Пользователь, создающий пространство, автоматически получает роль Менеджера.

- может выбирать рабочее пространство из списка доступных:

- видит только те пункты меню (левого и верхнего), вкладки и данные (объекты типов данных), которые относятся к выбранному рабочему пространству, например для **Service Desk**:

- видит объекты только тех типов данных, которые добавлены в данное пространство (а также «общие» типы данных глобального пространства);

**Возможности администратора:**

- управляет рабочими пространствами: создаёт, изменяет, удаляет, настраивает для них пункты меню и типы данных;
- задаёт принадлежность типов данных к **глобальному** пространству или к конкретному — это позволяет сделать объекты (например, общие справочники) доступными во всех рабочих пространствах одновременно;
- добавляет и исключает типы данных из рабочих пространств, разграничивая объекты между ними. Например, добавив тип «Договор» в пространства «Договоры компании А» и «Договоры компании Б», можно обеспечить полную изоляцию договоров этих компаний.

По умолчанию при развёртывании системы создаётся **Глобальное рабочее пространство** — оно открывается для пользователей по умолчанию. При необходимости администратор может назначить другое стартовое пространство через настройку конфигурации. См. :ref:`Рабочее пространство администратора<ws_admin>`.

Коробочные :ref:`функциональные модули<ecos_modules>` разбиты по рабочим пространствам для удобства навигации.
