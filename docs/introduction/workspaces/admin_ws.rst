Конфигурации рабочего пространства
-------------------------------------

Рабочеепространства, которое должно открываться по умолчанию
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Для пользователей по умолчанию открывается :ref:`персональное рабочее пространство<ws_personal>`. Но если есть потребность указать какое-то конкретное рабочее пространство (например, чтобы пользователи по умолчанию попадали на корпоративный портал), то это можно сделать через настройку **default-workspace**.

Перейдите в журнал :ref:`«Конфигурация ECOS»<configuration_admin>`. Журнал доступен по адресу: ``v2/admin?journalId=ecos-configs&type=JOURNAL``

Конфигурация **default-workspace**:

 .. image:: _static/default_setting_01.png
       :width: 600
       :align: center 

Укажите **id** рабочего пространства, которое должно открываться по умолчанию:

 .. image:: _static/default_setting_02.png
       :width: 400
       :align: center 

Включить/Выключить возможность создания рабочих пространств всем пользователям
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Конфигурация **workspaces-allow-create-for-all-users**, которая будет скрывать и не давать возможность создавать рабочие пространства. Например, на внешнем портале можно ограничить возможность создания рабочих пространств за ненадобностью.

 .. image:: _static/allow_ws_01.png
       :width: 600
       :align: center
