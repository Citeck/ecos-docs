Доступные рабочие пространства
--------------------------------------------------------------

Персональное рабочее пространство
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. _ws_personal:

Индивидуальное пространство на платформе предназначено для организации личной работы пользователя: управления задачами и документами, а также участия в бизнес-процессах. Пространство доступно каждому пользователю по умолчанию и не требует настройки со стороны администратора.

 .. image:: _static/02_personal_workspace_home.png
       :width: 700
       :align: center


В пространстве можно настроить :ref:`домашнюю страницу и меню<workspace_settings>`.

Задачи
"""""""

.. _tasks_spaces:

 .. image:: _static/02_personal_workspace.png
       :width: 700
       :align: center

Раздел **«Задачи»** содержит:

    -   **Текущие задачи** — все задачи, которые могут быть выполнены пользователем: как единственным исполнителем, так и в составе группы.
    -   **Завершенные задачи** — задачи, выполненные пользователем.
    -   **Задачи подчиненных** — задачи пользователей, у которых текущий пользователь указан в профиле как Линейный руководитель.

Библиотека документов
"""""""""""""""""""""

.. _ws_doc_library:

Представляет собой иерархический интерфейс для работы с папками и документами. Каждый пользователь видит только свои файлы и папки в рамках персонального пространства.

 .. image:: _static/05_doclib.png
       :width: 700
       :align: center

Загрузка файлов и папок доступна как по кнопке, так и перетаскиванием (drag & drop).

 .. image:: _static/05_doclib_01.png
       :width: 700
       :align: center

.. _ws_new_doc:

По кнопке **+** доступно добавление файлов и папок, а также создание новых документов:

 .. image:: _static/doc_01.png
       :width: 600
       :align: center

Выберите тип документа и укажите название:

 .. image:: _static/doc_02.png
       :width: 500
       :align: center

В OnlyOffice создаётся новый файл для совместной работы — над документом могут работать несколько пользователей одновременно:

 .. image:: _static/doc_04.png
       :width: 700
       :align: center

Подробно о функционале см. :ref:`Библиотека документов<document_library>`.

Доска задач
""""""""""""""

.. _ws_kanban:

**Доска задач** — наглядное представление персональных задач в виде виртуальной канбан-доски.

 .. image:: _static/06_kanban.png
       :width: 700
       :align: center

В колонках доски отображаются статусы задач, карточки задач распределены по столбцам. Нажав на наименование задачи, можно перейти в её карточку.

Переход задачи между статусами выполняется перемещением карточки по столбцам.

 .. image:: _static/06_kanban_01.png
       :width: 700
       :align: center

Для создания задачи нажмите **+**. Форма создания задачи аналогична форме создания поручения, но без полей **Исполнитель** и **Проверяющий** — задача всегда назначается на себя.

 .. image:: _static/06_kanban_02.png
       :width: 600
       :align: center

См. подробно о :ref:`Поручениях<ecos-assignments>`.


Совместное рабочее пространство
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. _ws_collaborative:

 .. image:: _static/joint_ws_01.png
       :width: 700
       :align: center

Совместное рабочее пространство предназначено для коллективной работы команд, отделов или проектных групп. Оно объединяет участников, документы, задачи и бизнес-процессы в едином интерфейсе. В отличие от персонального, содержимое пространства доступно всем его участникам.

Пользователь с ролью **Менеджер** может настроить :ref:`домашнюю страницу и меню<workspace_settings>`.

В **библиотеке документов** загруженные файлы и созданные документы доступны всем участникам пространства:

 .. image:: _static/joint_ws_02.png
       :width: 700
       :align: center

На **доске поручений** можно создавать поручения на участников пространства:

 .. image:: _static/joint_ws_03.png
       :width: 700
       :align: center

Функционал :ref:`поручений<ecos-assignments>` доступен в полном объёме.

Пользователь с ролью **Менеджер** может создавать :ref:`артефакты<ecos_artifacts>`, список которых представлен в разделе **Администрирование** рабочего пространства. См. :ref:`подробно<ws_artifacts>`.

О создании совместного рабочего пространства см. :ref:`Создание рабочего пространства<new_workspace>`.
